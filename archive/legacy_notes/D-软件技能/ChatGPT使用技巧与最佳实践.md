## 🎯 核心使用原则

### CLEAR原则
- **C**ontext: 提供充分的上下文信息
- **L**ength: 控制适当的输入和输出长度
- **E**xamples: 给出具体示例
- **A**udience: 明确目标受众
- **R**ole: 定义AI的角色

## 🚀 高效提示策略

### 1. 角色设定 (Role Playing)
```
你是一位资深的Python开发工程师，拥有10年以上的Web开发经验。
你的任务是帮我审查代码并提供优化建议。
```

**应用场景**:
- 专业咨询: 律师、医生、教师
- 创意写作: 编剧、诗人、小说家
- 技术指导: 工程师、设计师、分析师

### 2. 思维链提示 (Chain of Thought)
```
请一步步分析这个机器学习问题：

1. 首先分析数据特征
2. 然后选择合适的算法
3. 解释算法选择的原因
4. 预测可能的挑战
5. 提供解决方案
```

### 3. 示例学习 (Few-Shot Learning)
```
请按照以下格式转换文本：

输入: "用户登录失败"
输出: {"event": "login_failed", "severity": "warning", "category": "auth"}

输入: "数据库连接超时" 
输出: {"event": "db_timeout", "severity": "error", "category": "database"}

输入: "邮件发送成功"
输出:
```

## ⚠️ 注意事项与限制

### 内容准确性
- **数据截止**: 2024年4月 (具体版本而异)
- **实时信息**: 无法获取当前时间、股价、新闻
- **专业建议**: 不能替代专业医疗、法律、财务建议

### 隐私保护
```markdown
⚠️ 避免输入敏感信息：
- 个人身份信息 (姓名、地址、电话)
- 密码、API密钥、令牌
- 公司机密、商业秘密
- 个人隐私数据
```

### 内容政策
- 避免有害、非法、歧视性内容
- 不生成误导性信息
- 遵守版权和知识产权
